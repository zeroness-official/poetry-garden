# 🌍 2-city (ai / 人生)

ここは `poetry-garden` の `ai`（Application / 応用）層であり、「タビ（人生）」のメタファー を担当します。

## 役割

`0-garden`（NI層） と `1-village`（UI層） で完成した理論を、一般ユーザー（心の旅人）向けに「普及・実践」する「場」です。

あなたの「心の旅」はここから始まります（`departure`）。

## ai層としての機能

2-cityは**アプリケーション層**として、理論を日常生活に実装します：

- **離散化**：抽象的な原理を、具体的な行動ステップに分解
- **顕現**：言葉（理論）を、行動（現実）に変換
- **個別化**：一般理論を、個人の文脈に適用
- **フィードバック**：実践結果を1-village/0-gardenに還元

## ディレクトリ構造（心の旅の4段階）

```
2-city/
├── departure/   # 出立：気づき、動機、最初の一歩
├── trial/       # 試練：実践、失敗、学習
├── transform/   # 変容：深化、統合、悟り
└── return/      # 帰還：日常への統合、他者への貢献
```

この構造は「ヒーローズ・ジャーニー」（神話の法則）を反映しています。

## 実践原則

2-cityは**実装の場**です：

- **段階的**：無理なく、一歩ずつ
- **実験的**：正解を求めず、試してみる
- **個別的**：万人向けの解はない、自分の道を見つける
- **循環的**：一度通過したら終わりではなく、螺旋的に深まる

## 理論との関係

```
0-garden (ni)     → 原理の発見・理論構築
   ↓
1-village (ui)    → 解釈・翻訳・議論
   ↓
2-city (ai)       → 実践・適用・フィードバック
   ↓
0-garden (ni)     → 実践からの洞察が新たな理論を生む
```

2-cityは単なる消費者ではなく、**実践を通じた理論の検証と進化の場**です。

## メタファーの意味

**2-city = 人生の旅**

- 理論（0-garden）は「地図」
- 解釈（1-village）は「ガイドブック」
- 実践（2-city）は「実際の旅」

地図とガイドブックは旅の助けになるが、**実際に歩くのはあなた自身**です。2-cityは、その旅路を支援し、旅人同士が経験を分かち合う場です。

---

**aunixの実装**：あなたの人生という「ハードウェア」に、zeroness という「OS」をインストールし、新たな「アプリケーション（生き方）」を起動する場。それが2-cityです。
