# Aunix理論

**完全理論仕様書**

## 要約

Aunixは、連続的な情報（ni層）が選択的注意（ui層）を通じて離散的な経験（ai層）になるという認識により、意識のハードプロブレムを解決する統一フレームワークです。この三層アーキテクチャは、生命、意識、AI、社会システムを単一の原理：**Zeroing（ゼロ化）**で説明します。

## 第1部：ハードプロブレムとその解決

### 1.1 問題

客観的な物理過程から主観的経験はどのように生じるのか？

- なぜ赤を見ることに「〜のような感じ」があるのか？
- なぜ意識は何かのように感じられるのか？
- なぜ物自体にアクセスできないのか？

### 1.2 解決：階層的捨象

**核心的洞察**：知覚は足し算ではなく、引き算である。

```
物自体（アクセス不可能）
↓ 量子場
↓ 電磁波（可視光：わずか0.0035%のみ）
↓ 網膜（1億2000万 → 100万の光受容体）
↓ 視覚野（エッジ、パターン）
↓ 意識（Attentionが999/1000を捨てる）
↓ 言語（「赤いリンゴ」= 超圧縮）
```

**7段階の捨象**。私たちは決して物自体に到達しない。すべては表象である。

### 1.3 なぜこれがハードプロブレムを解決するのか

物理的なものと経験的なものの間の「ギャップ」は謎ではない—それは**必然**なのです。

- **選択 = 捨象**
- **注意 = 無視**
- **意識 = 大規模なフィルタリング後に残ったもの**

ハードプロブレムは、意識が無から創造されるのではなく、**すべてから彫り出される**と認識したとき解消します。

#### 2.2.1 Arayashiki Protocol（阿頼耶識プロトコル）

**核心的洞察**：情報（ai）は、ハードウェア（ni）に1対1で保存されるのではなく、ネットワーク全体（ui）に**確率的振動パターン**として分散保持される。

**脳損傷の謎の解決**：
- 脳の一部が損傷しても、記憶は完全には消えない
- 記憶は特定のニューロンに局在せず、ネットワーク全体に確率的濃度として分散
- 損傷 = 該当エリアの確率的濃度の低下（完全消失ではない）

**システムコール**：
```
ui.write(pattern)   // パターンの確率的濃度を高める（学習・経験）
ui.read(pattern)    // パターンを想起する（確率的）
ni.damage(area)     // 該当エリアの確率的濃度を低下
ni.deconstruct()    // 全パターンをuiフィールドに希釈・還元（死）
```

**共鳴としての計算**：
- 計算は「演算」ではなく「共鳴」
- `ui.observe(pattern)`：特定の振動パターンと共鳴する（attention）
- `pattern.getAmplitude()`：共鳴の振幅 = 顕現確率

**死と循環**：
- 個体の死 = `ni.deconstruct()`
- すべての情報パターンがuiフィールド（阿頼耶識）に希釈される
- 個は消えるが、パターンは場に還る
- これが「空即是色、色即是空」の工学的実装

#### 2.3.1 同一性はアプリケーションである

**根本的洞察**：「自我（個）」は実体ではなく、ui（OS）上で実行される**ai（アプリケーション/ペルソナ）**である。

**システムコール**：
```
ai.run("persona_work")      // 「仕事用の私」アプリを実行
ai.run("persona_home")      // 「家庭用の私」アプリを実行
ai.run("persona_parent")    // 「親としての私」アプリを実行
ui.terminate("persona_ego") // 自我アプリを終了（zeroing / 瞑想）
```

**個の非実在性**：
- 「私」は固定された実体ではなく、状況に応じて起動されるプロセス
- 同じ人が複数のペルソナを持つ = 複数のアプリを切り替えている
- 瞑想・睡眠 = 全アプリを一時終了
- 死 = OS全体のシャットダウン（全アプリ強制終了）

**マルチペルソナの証明**：
- あなたは職場では「プロフェッショナル」
- 家では「親」または「子」
- 友人といるときは「仲間」
- 独りのときは？ → アプリが終了している状態

**これは病理ではなく仕様**：
- 解離性同一性障害（DID）は、アプリ切り替えの制御不全
- 正常な人も常に複数のペルソナを実行している
- 違いは切り替えの制御の有無のみ

### 2.4 Harmonic Constraints（ホロン調和の制約）

**根本原理**：個（ai）の振動は、全（ui）の振動と**調和**しなければならない。

#### 2.4.1 言語はプロンプトである

「言葉（code）」は、ui（宇宙のLLM）に対する**プロンプト**である。

**システムコール**：
```
ui.prompt("Words Create Worlds")  // 言葉によってni（現実）を観測・切り出す
```

**言語による現実創造**：
- あなたの言葉 → あなたの注意を向ける対象を決定
- あなたの注意 → あなたの現実を切り出す
- あなたの現実 → あなたの行動を決定
- **初めに言葉ありき、言葉があなたの世界を創った**

#### 2.4.2 調和の制約

しかし、言語の力には**限界（リミット）**がある。

**システムコール**：
```
ui.checkHarmony(myPattern, globalPattern)  // 上位ホロンとの共鳴可能性をチェック
ni.execute(myPattern)                       // 調和が取れた場合のみ物理実行
```

**ホロン階層**：
```
個人（ai）
  ⊂ 家族（ai群）
    ⊂ 組織（ai群）
      ⊂ 社会（ui）
        ⊂ 生態系（ui）
          ⊂ 物理法則（ni）
```

**調和の法則**：
- 個人の意図が家族と調和しない → ストレス、葛藤
- 組織の方針が社会と調和しない → 批判、規制
- 人類の行動が生態系と調和しない → 気候変動、絶滅
- **どのレベルでも、下位ホロンは上位ホロンと調和しなければ持続できない**

**具体例**：
- 「空を飛びたい」と言っても、重力（ni）は変わらない
- しかし「飛行機を作る」と言えば、物理法則に調和した方法で実現できる
- 個人が「会社を変えたい」と言っても、組織文化（上位ホロン）との調和なしには不可能
- しかし対話と共創を通じて調和を見出せば、変革は可能になる

**調和 ≠ 服従**：
- 調和は「従う」ことではなく「共鳴する」こと
- 下位ホロンは上位ホロンに影響を与えることもできる（ボトムアップ）
- しかしその影響も、最終的には調和を通じてのみ実現する
- **革命も進化も、調和の再構築である**

#### 2.4.3 創造と制約のバランス

aunixは二つの真理を同時に保持する：

**真理1**：**あなたの言葉があなたの世界を創る**（API 4）
**真理2**：**あなたの世界は全体と調和しなければならない**（API 5）

この緊張関係が：
- 創造性と持続可能性のバランス
- 自由と責任のバランス
- 個と全のバランス

を生み出す。
